# Flutter App - Agent Template

A Flutter mobile/web application with AI chat capabilities, built with Riverpod and Shadcn UI.

## Project Structure

```
lib/
├── main.dart                    # App entry point
├── firebase_options.dart        # Firebase configuration
├── core/
│   ├── constants/              # API endpoints, constants
│   ├── services/               # Analytics, Remote Config services
│   ├── theme/                  # Theme configuration
│   ├── utils/                  # Utilities (logging, etc.)
│   └── widgets/                # Shared widgets
├── data/
│   ├── models/                 # Data models (generated from DB)
│   ├── providers/              # Riverpod providers
│   ├── repositories/           # Data access layer
│   └── services/               # Business logic services
├── features/
│   ├── auth/                   # Authentication (login, etc.)
│   ├── chat/                   # Chat UI and AI providers
│   ├── home/                   # Home screen
│   └── profile/                # User profile, settings
└── routing/                    # GoRouter configuration
```

## Key Dependencies

- **State Management**: Riverpod (with code generation)
- **Navigation**: GoRouter
- **UI**: Shadcn Flutter (Material-based)
- **Backend**: Supabase (Auth, Database, Edge Functions)
- **AI**: Flutter AI Toolkit with custom providers

## Setup

1. Copy `.env.example` to `.env`:
   ```bash
   cp .env.example .env
   ```

2. Fill in your credentials:
   - `SUPABASE_URL` - from Supabase project settings
   - `SUPABASE_ANON_KEY` - from Supabase project settings

3. Install dependencies:
   ```bash
   flutter pub get
   ```

4. Run the app:
   ```bash
   flutter run
   ```

## Features

### Authentication
- Firebase Auth (Google, Apple sign-in)
- Supabase Auth integration
- Auth state managed via Riverpod

### AI Chat
- Multiple AI provider support (Hermes, Firebase AI)
- Provider switching via `AIProviderFactory`
- Conversation history with ratings
- Tool execution support

### Analytics
- Firebase Analytics integration
- AI usage tracking
- Performance metrics dashboard

## Adding New AI Providers

1. Create a provider class extending `AIChatProvider`:
   ```dart
   class MyProvider extends AIChatProvider { ... }
   ```

2. Register in `AIProviderFactory`:
   ```dart
   case AIProviderType.myProvider:
     return MyProvider(...);
   ```

3. Add provider option to UI

## Generated Code

Some files are auto-generated and should not be edited manually:
- `lib/data/models/generated/*.dart` - Generated from database schema
- `*.g.dart` files - Generated by Riverpod

To regenerate:
```bash
# Database models (requires Supabase DB)
cd supabase/db && npm run gen:dart

# Riverpod providers
cd flutter && flutter pub run build_runner build --delete-conflicting-outputs
```
